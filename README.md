# s485s StrGenUrl

Chrome-расширение для быстрого формирования URL и произвольных строк по шаблонам.

---

## Что это и зачем

Если вы регулярно открываете одни и те же URL с разными параметрами — например, страницы разных окружений (`dev`, `staging`, `prod`) или с разными идентификаторами — это расширение избавит от ручной правки адресов.

Вы один раз описываете шаблоны и переменные в конфиге, а дальше просто выбираете нужные значения из выпадающих списков и копируете или открываете готовую строку.

---

## Установка

1. Скачайте или клонируйте репозиторий.
2. Откройте Chrome → адресная строка → `chrome://extensions/`
3. Включите **Developer mode** (переключатель в правом верхнем углу).
4. Нажмите **Load unpacked** и выберите папку с расширением.
5. Нажмите на иконку расширения в тулбаре — откроется боковая панель.

---

## Как пользоваться

### Боковая панель

Открывается кнопкой расширения. Здесь происходит вся основная работа:

- Выбирайте значения переменных из выпадающих списков.
- Результирующая строка обновляется мгновенно.
- **Copy** — скопировать строку в буфер обмена.
- **Open** — открыть строку как URL в новой вкладке (доступно только для шаблонов с типом `url`).

Незаполненные плейсхолдеры подсвечиваются оранжевым — это подсказка, что переменная ещё не выбрана.

### Страница настроек

Открывается кнопкой **Settings** в боковой панели. Здесь вы редактируете конфиг в формате YAML.

После нажатия **Save** конфиг валидируется и сохраняется. Боковая панель обновляется автоматически, без перезагрузки.

Кнопка **Reset to Default** возвращает конфиг к встроенному примеру.

## Формат конфига

Конфиг — это YAML-документ. Верхний уровень — список проектов.

```yaml
projects:
  - name: My Project          # название проекта (обязательно)
    description: optional     # подпись под названием (необязательно)
    envs:
      - name: "host"          # имя переменной — используется в шаблоне как {host}
        description: optional # подпись (необязательно)
        values:               # список значений в выпадающем списке
          - example.com
          - staging.example.com
    patterns:
      - name: Main page       # название шаблона (необязательно)
        pattern: "https://{host}/dashboard"   # шаблон со вставками {имя_переменной}
        type: url             # тип: url (появляется кнопка Open) или str
```

### Пример из жизни

```yaml
projects:
  - name: Backend API
    description: REST сервис
    envs:
      - name: env
        values:
          - api.example.com
          - staging-api.example.com
          - localhost:8080
      - name: userId
        values:
          - 1001
          - 1002
          - 9999
    patterns:
      - pattern: "https://{env}/users/{userId}"
        type: url
        name: User profile
      - pattern: "https://{env}/users/{userId}/orders"
        type: url
        name: User orders
      - pattern: "{env}/users/{userId}"
        type: str
        name: Path only (for curl)
```

Выбираете `env` и `userId` — и сразу видите готовые ссылки для всех шаблонов.

---

## Хранение данных

Конфиг сохраняется в `chrome.storage.sync` — синхронизируется между всеми Chrome-браузерами, где вы вошли в аккаунт Google. Никаких внешних серверов и передачи данных нет.
